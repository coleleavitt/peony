# Theorem-to-Rust Bridge Map

This table is the theorem-to-Rust bridge inventory. It records the current Rust
surface, test gate, and trusted boundary for each theorem family used in public
verification claims. The public status table lives in
[`docs/VERIFICATION_CLAIMS.md`](VERIFICATION_CLAIMS.md), with the
machine-readable source in [`docs/VERIFICATION_CLAIMS.json`](VERIFICATION_CLAIMS.json).
`implementation-bridge` here means the theorem has a checked Rust bridge surface
for the named scope. It is not whole-linker correctness.

## P1 Proof-Assumption Audit Hook

P1 extends this inventory through
[`docs/VERIFICATION_CLAIMS.json`](VERIFICATION_CLAIMS.json) and
[`docs/verification-assumptions/`](verification-assumptions/). The bridge gate
checks that every theorem used by a `theorem-bridged` or
`implementation-verified` public claim has a `Print Assumptions` artifact, and
that the artifact is either closed under the global context or uses only the
standard functional-extensionality axiom already named by the Rocq README.
